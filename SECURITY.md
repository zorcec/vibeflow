# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest  | ✅ |
| Older   | ❌ Please upgrade |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Report security issues privately by emailing: **vibeflow.tools@gmail.com**

Please include:
- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

You will receive a response within 48 hours. We will work with you to understand the issue, develop a fix, and coordinate a disclosure timeline.

## Scope

This policy covers the CLI tool (`@vibeflow-tools/cli`) and shared packages in this repository. The SaaS web application at [app.vibeflow.tools](https://app.vibeflow.tools) is outside the scope of this repository.

## Responsible Disclosure

We ask that you:
- Give us reasonable time to fix the issue before public disclosure
- Not exploit the vulnerability beyond what is necessary to demonstrate it
- Not share the vulnerability with others until it has been fixed
